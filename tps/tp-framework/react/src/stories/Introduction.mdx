## 2️⃣ Structure d’un composant React

Un composant React moderne suit généralement cette architecture :

```jsx
// 1. Importations
import { useState } from "react";
import "./Button.css";

// 2. Déclaration du composant
export default function Button({ label }) {
  // 3. Hooks et logique interne
  const [count, setCount] = useState(0);

  // 4. Fonctions internes
  const handleClick = () => {
    setCount(count + 1);
  };

  // 5. Rendu JSX
  return (
    <button className="btn" onClick={handleClick}>
      {label} — {count}
    </button>
  );
}
```

## Points clés

- **Imports** : toujours en haut du fichier.
- **Composant** : `function MonComposant(props)` ou `const MonComposant = () => {}`.
- **Hooks** : `useState`, `useEffect`, etc.
- **Fonctions internes** : logique spécifique au composant.
- **JSX** : ce que le composant affiche à l’écran.

## Bonnes pratiques

- Un **composant = un fichier**.
- Le **nom du composant = nom du fichier**.
- CSS **spécifique au composant** pour éviter les conflits.
- Favoriser la **réutilisation et la modularité**.
