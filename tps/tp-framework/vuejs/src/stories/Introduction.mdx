## 2️⃣ Structure d’un composant Vue 3

Un composant Vue moderne suit généralement cette architecture :

```vue
<template>
  <button @click="increment">{{ label }} — {{ count }}</button>
</template>

<script setup>
import { ref } from 'vue';
import './Button.css';

defineProps({
  label: String
});

const count = ref(0);

const increment = () => {
  count.value++;
};
</script>

<style scoped>
button {
  padding: 0.5rem 1rem;
  background-color: #42b983;
  color: white;
  border: none;
  border-radius: 5px;
  cursor: pointer;
}
</style>
```

## Points clés

- **Template** : structure HTML affichée.
- **Props** : pour transmettre des données au composant.
- **State réactif** : via `ref` ou `reactive`.
- **Fonctions internes** : logique spécifique au composant.
- **Composition API** : `setup()`, `ref`, `reactive`, `computed`.

## Bonnes pratiques

- Un **composant = un fichier `.vue`**.
- Le **nom du composant = nom du fichier**.
- CSS **scopé** pour éviter les conflits (`<style scoped>`).
- Favoriser la **réutilisation et la modularité**.
